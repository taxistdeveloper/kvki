## 1. Дизайн-концепция

### Философия

Сайт колледжа — это **точка входа в образовательную экосистему**. Пользователи приходят с разными целями: абитуриенты ищут информацию о поступлении, студенты — расписание и документы, родители — доверие к учреждению. Дизайн должен быть:

- **Образовательным** — визуальный язык подчёркивает серьёзность, профессионализм, академическую среду
- **Современным** — без «казённости», с аккуратными формами и актуальными паттернами
- **Лёгким** — много воздуха, не перегруженные блоки, быстрая загрузка
- **Интуитивным** — пользователь понимает, куда нажать, без обучения
- **Доверительным** — цвета, типографика и структура вызывают ощущение надёжности

### Ключевые принципы

| Принцип | Реализация |
|---------|------------|
| **Ясность** | Чёткая иерархия, один главный CTA на экран |
| **Доступность** | Контраст 4.5:1+, читаемые размеры шрифтов, фокус-состояния |
| **Консистентность** | Единые отступы, кнопки, карточки во всём интерфейсе |
| **Прогрессивное раскрытие** | Сложное меню не перегружает, раскрывается по необходимости |
| **Ориентация на действие** | CTA заметны, формы короткие, путь к цели — короткий |

### Визуальный тон

- **Сдержанный, но не холодный** — тёплые нейтрали (slate, stone) вместо чистого серого
- **Акцент — синий/бирюзовый** — ассоциация с образованием, технологиями, надёжностью
- **Минимум декора** — геометрия, иконки, аккуратные тени вместо иллюстраций

## 7. Mobile Navigation Behavior

### Принципы

- **Thumb zone** — важные элементы в зоне досягаемости большого пальца
- **Один тап** — основное действие доступно без раскрытия меню
- **Прогрессивное раскрытие** — вложенные уровни открываются по требованию

### Поведение

```
1. Иконка «гамбургер» (три полоски) в header
   └── Заменяет десктопное меню на экранах < 1280px

2. Тап по иконке
   └── Открывается full-screen или slide-in панель меню
   └── Оверлей затемняет контент (opacity 0.5)
   └── Кнопка превращается в «крестик» (закрыть)

3. Структура мобильного меню
   ├── Пункты 1-го уровня — крупные, 48px высота (touch target)
   ├── Пункты с подменю — стрелка вправо
   ├── Тап по пункту с подменю
   │   └── Раскрытие вложенного списка (accordion)
   │   └── Отступ вложенных: 16px
   ├── Тап по конечному пункту
   │   └── Переход + закрытие меню
   └── Свайп вниз или тап по оверлею — закрытие

4. Sticky header
   └── Высота 56–64px
   └── Logo + hamburger
   └── При скролле вниз — можно скрывать (опционально)
```

### Рекомендуемые размеры touch targets

- Минимум 44×44px для интерактивных элементов
- Пункты меню: 48px высота, 16px padding по горизонтали

## 8. UX рекомендации

### Общие

1. **Сократить путь к цели** — абитуриент должен за 2–3 клика найти правила приёма
2. **Ясные CTA** — «Подать заявку», «Узнать больше», не «Подробнее» везде
3. **Контекстная помощь** — подсказки рядом с формами, FAQ по разделам
4. **Обратная связь** — загрузка, успех, ошибки должны быть видны

### Для абитуриентов

- Отдельный раздел «Абитуриентам» с подразделами: Информация, Правила приёма, Специальности
- Дедлайны и даты — крупно, с иконкой календаря
- Контакты приёмной комиссии — на видном месте
- Возможность скачать документы (PDF)

### Для студентов

- Быстрый доступ к расписанию, SKILLS-паспорту, кружкам
- Центр обслуживания — один вход для всех сервисов
- Уведомления о важных событиях

### Доступность

- Контраст текста минимум 4.5:1 (WCAG AA)
- Фокус-состояния для клавиатурной навигации (ring 2px)
- Alt-тексты для изображений
- Семантическая разметка (header, main, nav, footer)

### Производительность

- Lazy load изображений
- Минимум внешних скриптов
- Critical CSS inline для первого экрана

## 9. CSS Architecture

### Подход

- **Utility-first (Tailwind)** — быстрая вёрстка, консистентность
- **Компонентные классы** — для повторяющихся паттернов
- **CSS-переменные** — для темизации и переопределения

### Структура файлов

```
assets/css/
├── base.css        # Reset, переменные, типографика
├── components.css  # Кнопки, карточки, inputs, навигация
├── layout.css      # Header, footer, grid, spacing
├── utilities.css   # Дополнительные утилиты
└── style.css       # Импорты, переопределения Tailwind
```
